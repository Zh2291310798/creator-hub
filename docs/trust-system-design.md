# CreatorHub 信任体系设计

> **版本**: v1.0 | **日期**: 2026-06-03 | **作者**: P10/CTO
> **问题**: 品牌方的核心痛点（假数据创作者）没解决。自填数据 = 零信任。

---

## 一、信任分层

信任不是单一功能，是逐层建立的：

```
Layer 4: 交易评价         ← 交易完成后互评, 累计信誉
Layer 3: 平台认证         ← 连接真实平台账号, 拉取公开数据
Layer 2: 社区社交资本     ← 等级/帖子/好友/活跃度 (已有 ✅)
Layer 1: 身份基准         ← 实名/企业认证 (v2.0)
```

Layer 2 已有。现在补 Layer 3（平台认证）和 Layer 4（交易评价）。

---

## 二、Layer 3：平台账号连接 + 数据验证

### 设计原则
- **不做 OAuth**（太重要求创作者授权）
- **只拉公开数据**（粉丝数/作品数/互动量，平台公开页就能抓）
- **创作者主动提交**（填平台主页 URL，系统验证后展示）

### 用户体验流

```
个人主页 → "连接平台账号" →
  ┌──────────────────────────────────────┐
  │ 选择平台: [小红书] [抖音] [B站] ...   │
  │                                      │
  │ 输入主页链接: __________________      │
  │ 例如: https://xiaohongshu.com/xxx    │
  │                                      │
  │ [验证]  ← 拉取公开数据               │
  │                                      │
  │ ┌─ 验证结果 ─────────────────────┐   │
  │ │ 粉丝: 2.3万  ✅ 已确认          │   │
  │ │ 作品: 47 篇                     │   │
  │ │ 获赞: 12.8万                    │   │
  │ │ 平台: 小红书                    │   │
  │ │ 最后更新: 2026-06-03            │   │
  │ │ 认证标识: 🏅 已连接             │   │
  │ └────────────────────────────────┘   │
  │                                      │
  │ [保存并展示]                          │
  └──────────────────────────────────────┘
```

### MVP 实现方案

**不建后端爬虫**。用两步替代：

**Step 1: URL 格式校验**（前端）
```js
const PLATFORM_PATTERNS = {
  xhs: /xiaohongshu\.com\/explore\/|xhslink\.com/,
  douyin: /douyin\.com\/user\//,
  bilibili: /bilibili\.com\/space\//,
  kuaishou: /kuaishou\.com\/profile\//,
  weibo: /weibo\.com\/u\//,
};
```

**Step 2: 创作者手动输入关键数据 + 截图上传（v2.0 再接入 API）**
- 粉丝数（必填）
- 作品数（选填）
- 平台认证状态（是否蓝V/黄V/企业号）
- 截图（防篡改的社交证明）

**显示逻辑**：
- 只提交 URL → 显示 🔗 已连接（弱信任）
- URL + 数据 + 截图 → 显示 ✅ 已验证（中信任）
- 后续接入 API 自动拉取 → 显示 🏅 官方验证（强信任）

### 数据模型

```sql
ALTER TABLE profiles ADD COLUMN IF NOT EXISTS connected_platforms JSONB DEFAULT '[]';
-- [{platform:'xhs', url:'...', followers:23000, verified:true, screenshot_url:'...', verified_at:'...'}]
```

### 对品牌方的价值

对接广场的达人卡片变为：

```
┌──────────────────────────┐
│ 🏅 已连接            张小美 │
│ 📍 上海                  │
│ 📕 小红书 2.3万粉 ✅     │
│ 🎬 抖音 8.7万粉 ✅       │
│ Lv.7 资深创作者          │
│ [查看主页] [留言意向]     │
└──────────────────────────┘
```

品牌方一眼看到：这个达人的数据**连接了真实平台**，不是瞎填的。

## 三、Layer 4：交易评价体系

### 评价时机

```
对接流程:
  品牌发布需求 → 达人留言意向 → 双方聊天确认 →
  [开始合作] → 合作完成 →
  🆕 双方互评 (品牌评价达人 / 达人评价品牌)
```

### 评价维度

**品牌评价达人（5 星 × 4 维度）**：
| 维度 | 说明 |
|------|------|
| 内容质量 | 产出的内容是否符合预期 |
| 沟通效率 | 响应速度、配合度 |
| 专业度 | 是否按时交付、是否主动解决问题 |
| 数据表现 | 实际曝光/互动是否达标 |

**达人评价品牌（5 星 × 3 维度）**：
| 维度 | 说明 |
|------|------|
| 需求清晰度 | Brief 是否明确 |
| 付款及时性 | 是否按时结算 |
| 合作体验 | 是否尊重创作者 |

### 展示规则

```
达人个人主页:
  ⭐ 4.8 (12 次合作)
  "内容质量高，沟通顺畅" — 某品牌主理人
  "非常专业，下次还会合作" — 某MCN机构

品牌主页 (新增):
  ⭐ 4.5 (8 次合作)
  "Brief 清晰，付款及时" — 某达人
```

### 防刷机制（MVP 够用）
- 只有双方在聊天中有 ≥ 5 条消息才能互评（防止机器人刷评）
- 评价内容走 escapeHtml + 敏感词过滤
- 差评（≤2 星）需填写具体原因（≥20 字）
- 评价不可修改、不可删除（防止报复性删评）

### 数据模型

```sql
CREATE TABLE IF NOT EXISTS reviews (
  id BIGSERIAL PRIMARY KEY,
  deal_id TEXT NOT NULL,           -- 关联的对接需求 id
  reviewer TEXT NOT NULL,          -- 评价人 username
  reviewee TEXT NOT NULL,         -- 被评价人 username
  role TEXT NOT NULL,             -- 'brand_review_creator' | 'creator_review_brand'
  ratings JSONB NOT NULL,         -- {content_quality:5, communication:4, ...}
  comment TEXT DEFAULT '',
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```

## 四、信任等级体系

整合所有信任信号，给创作者一个"信任分"：

```
信任分 = 
  社区社交资本 (30%): 等级 × 10 + 帖子数 × 2 + 好友数
  平台认证 (40%):   每个已连接平台 +15 分, 有截图的 +10 分
  交易评价 (30%):   平均评星 × 10 × 评价数 (最少 5 条起算)

信任等级:
  0-50:   🌱 新人
  51-150: 🌿 成长中
  151-300: 🌳 值得信赖
  301+:   👑 顶级信誉
```

**展示位置**：达人卡片、个人主页、对接搜索结果排序（默认按信任分降序）
