# CreatorHub 产品需求文档 (PRD)

> **版本**: v1.1 | **日期**: 2026-06-02 | **阶段**: 2 — 需求与数据定义
> **基于**: 远景文档 v1.0 + 用户洞察 v2.0 + MVP 假设 v1.0 + 原型 v1.0

---

## 一、产品概述

CreatorHub 是中国市场跨平台创作者对接社区。核心差异化：**先归属，后交易**——用社区（等级/好友/帖子）建立信任，再用对接（品牌⇄达人/本地匹配/招募）完成交易。

## 二、用户故事

### Epic 1：新人入行 (H4)

| ID | 用户故事 | 验收标准 | 优先级 |
|----|---------|---------|:---:|
| US1.1 | 作为刚毕业的大学生，我想在注册时标注"学生/新人"身份，让平台给我推荐适合的入门内容 | 注册表单新增 2 个身份选项；注册后跳转新手引导页 | P1 |
| US1.2 | 作为新人，我想在首次登录时看到入门推荐的帖子和前辈，知道从哪里开始 | 新手首页展示：入门文章 ≥3 篇 + 推荐前辈 ≥5 人 + 热门话题 ≥3 个 | P1 |
| US1.3 | 作为新人，我想在社区发"新人报道"帖，和同行建立第一次连接 | 社区新增"新人报到"分类；首次发帖 +50 XP | P2 |

### Epic 2：品牌↔达人对接 (H2)

| ID | 用户故事 | 验收标准 | 优先级 |
|----|---------|---------|:---:|
| US2.1 | 作为品牌方，我想按平台/类目/预算筛选达人，快速找到匹配的合作对象 | 对接页筛选器包含：平台、类目、粉丝量区间、报价区间 | P0 |
| US2.2 | 作为达人，我想在一个地方看到所有品牌的合作需求，不用在各平台之间来回刷 | 达人视图下列出所有品牌方发布的对接需求 | P0 |
| US2.3 | 作为品牌方/达人，我想对感兴趣的对象留言表达合作意向 | 点击"留言意向"弹出输入框，发送后对方在聊天中收到通知 | P0 |

## 四、优先级矩阵

```
重要 ┤  P1 快速迭代          P0 立即做
     │  新手引导完整流程      品牌↔达人对接优化
     │  行业讨论分类          本地匹配核心功能
     │  岗位/工作模式标签      城市筛选
     │
     │  P3 以后              P2 这个版本
     │  帖子搜索              新人报到分类
     │  商家入驻入口          好友筛选增强
     │  消息持久化增强        经验明细记录
     └─────────────────────────────
       不紧急                  紧急
```

## 五、非功能需求

| 类别 | 需求 | 指标 |
|------|------|------|
| 性能 | 首屏加载 | ≤ 2 秒 |
| 性能 | 页面切换 | ≤ 200ms |
| 兼容性 | 浏览器 | Chrome/Firefox/Safari/Edge 最近 2 个大版本 |
| 响应式 | 移动端适配 | 768px+ 全功能，375px+ 可浏览 |
| 数据安全 | 密码存储 | localStorage 本地存储（MVP 阶段，后续迁移后端加密） |
| 可维护性 | 代码结构 | 单文件 ≤ 5000 行，超限后拆分模块 |
