# CreatorHub 测试方案

> **版本**: v1.0 | **日期**: 2026-06-03 | **阶段**: 5 — 测试与质量保障

---

## 一、测试策略概览

| 维度 | 策略 | 工具 |
|------|------|------|
| 功能测试 | 手工场景化测试 + 检查清单 | 浏览器 DevTools |
| 语法验证 | 每次修改后 `node --check` | Node.js |
| 数据完整性 | localStorage 读写校验 | DevTools Console |
| 埋点验证 | 事件触发 → localStorage 检查 | Console + Application tab |
| 兼容性 | 主流浏览器手动验证 | Chrome / Edge / Firefox |
| 性能 | Lighthouse + 手动计时 | Chrome DevTools |
| 安全 | 手动 XSS 检查 + CSP | 代码审查 |

> **说明**: CreatorHub 是单文件 Vanilla JS SPA，无框架、无构建工具。测试以手工场景验证为主，自动化仅覆盖语法检查和 CI 结构检查。

## 五、回归测试检查清单

- [ ] `node --check` 通过（语法无错误）
- [ ] 登录/注册正常
- [ ] 社区发帖+评论正常
- [ ] 对接浏览+筛选+意向发送正常
- [ ] 本地匹配浏览+发布+删除+邀约正常
- [ ] 招聘浏览+发布正常
- [ ] 聊天私聊+世界频道正常
- [ ] 通知铃铛+红点+跳转正常
- [ ] 好友添加+查看主页正常
- [ ] 设置修改+保存正常
- [ ] Toast 提示正常弹出

## 六、测试数据准备

```js
// 在 Console 中运行——生成测试数据
(function() {
  // 创建测试用户
  const testUsers = {
    'test_creator': { password: '123456', role: 'creator', city: '北京' },
    'test_brand': { password: '123456', role: 'brand', city: '上海' },
    'test_student': { password: '123456', role: 'student', city: '广州' },
    'test_live': { password: '123456', role: 'freelancer', city: '深圳' },
    'test_media': { password: '123456', role: 'career_switcher', city: '杭州' },
  };
  localStorage.setItem('creatorhub_users', JSON.stringify(testUsers));
  console.log('✅ 5 个测试用户已创建');
  
  // 查看埋点数据
  const events = JSON.parse(localStorage.getItem('creatorhub_events') || '[]');
  console.log('📊 埋点事件数:', events.length);
  console.table(events.slice(-20).map(e => ({ event: e.event, user: e.user, time: new Date(e.timestamp).toLocaleTimeString() })));
})();
```
