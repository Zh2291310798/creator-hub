## 六、部署步骤

### GitHub Pages（自动）
```bash
git add index.html
git commit -m "release: v1.0.0 MVP"
git push origin main
# CI 自动部署 → Settings > Pages 查看 URL
```

### Vercel（手动首次，后续自动）
```bash
# 首次: vercel CLI 或 Dashboard 导入 GitHub 仓库
# 后续: 每次 push 自动部署
```

### 部署后验证
```bash
# 1. 检查页面可访问
curl -I https://<domain>/

# 2. 检查 Cache-Control
curl -I https://<domain>/ | grep Cache-Control
# 预期: Cache-Control: public, max-age=3600

# 3. 检查文件大小
curl -s https://<domain>/ | wc -c
```

## 七、回滚方案

| 场景 | 操作 | 恢复时间 |
|------|------|:---:|
| P0 Bug（登录不可用） | `git revert` → push | < 2 min |
| 白屏 | Vercel Dashboard → 回滚到上一个 Deploy | < 1 min |
| 数据丢失 | 无法回滚（localStorage 在用户端） | — |
| XSS 漏洞 | 紧急修复 → `git push` | < 30 min |
